SFML: перший інтерактивний проєкт на C++

На цьому занятті ми створюємо невеликий графічний застосунок: кольорова кулька рухається у вікні, відбивається від стін, змінює колір, реагує на клавіатуру і мишку.

Головна ідея заняття: графічна програма працює в циклі. У цьому циклі ми читаємо події, оновлюємо стан і малюємо кадр.

1. Що таке SFML

SFML — це бібліотека для створення простих графічних програм. Вона допомагає:

  • створювати вікна;
  • малювати фігури, текст, зображення;
  • читати події клавіатури і миші;
  • працювати з часом, звуком та іншими елементами.

Ми використовуємо готовий шаблонний проєкт, тому можемо зосередитися на коді програми.

Для більшої інформації про SFML, відвідайте офіційний сайт і сторінку документації.

2. Як відкрити шаблонний проєкт

  1. Клонуйте репозиторій з шаблоном.
  2. Відкрийте папку проєкту у VS Code.
  3. Основний код знаходиться у файлі src/main.cpp.
  4. Запуск проєкту виконується через F5.

У шаблоні вже підготовлені налаштування збірки, тому вручну прописувати довгі команди не потрібно.

3. Чим графічна програма відрізняється від консольної

Консольна програма

Часто працює так: прочитали вхідні дані, виконали обчислення, вивели відповідь, завершилися.

Графічна програма

Створює вікно і працює постійно: чекає подій користувача та перемальовує екран багато разів на секунду.

while (window.isOpen()) {
    // 1. Обробка подій
    // 2. Оновлення стану
    // 3. Малювання кадру
}

4. Вікно і головний цикл

Спочатку створюємо вікно. Потім, поки воно відкрите, програма повторює свій головний цикл.

#include <SFML/Graphics.hpp>
#include <optional>

int main() {
    sf::RenderWindow window(sf::VideoMode({800, 600}), "SFML lesson");
    window.setFramerateLimit(60);

    while (window.isOpen()) {
        while (const std::optional event = window.pollEvent()) {
            if (event->is<sf::Event::Closed>()) {
                window.close();
            }
        }

        window.clear(sf::Color::Black);
        window.display();
    }
}
Якщо не обробити подію закриття вікна, кнопка закриття може не спрацювати так, як очікується.

5. Малювання кадру

Кожен кадр будується заново:

  1. очищаємо екран;
  2. малюємо всі об’єкти;
  3. показуємо готовий кадр.
window.clear(sf::Color(30, 30, 30));
window.draw(ball);
window.display();

Якщо не викликати display(), новий кадр не буде показаний. Якщо не викликати clear(), малюнок може «накладатися» на попередній кадр.

6. Події, клавіатура і мишка

Треба розрізняти події і поточний стан.

  • Подія — це разова дія: натиснули кнопку, клікнули мишкою, закрили вікно.
  • Поточний стан — це те, що відбувається зараз: клавіша затиснута або ні.
// Разова подія
while (const std::optional event = window.pollEvent()) {
    if (event->is<sf::Event::Closed>()) {
        window.close();
    }
}

// Безперервне керування
if (sf::Keyboard::isKeyPressed(sf::Keyboard::Key::Left)) {
    // рух вліво
}
Зручно пам’ятати так: події — для разових реакцій, перевірка стану — для безперервного руху.

7. Стан об’єкта

Щоб кулька «жила», у неї мають бути змінні стану:

  • позиція;
  • швидкість;
  • радіус;
  • колір.

Кулька рухається не тому, що є спеціальна команда «рухатись», а тому що ми кожного кадру змінюємо її координати.

8. Рух і прив’язка до часу

Поганий спосіб — рухати об’єкт на однакову кількість пікселів за кадр. На різних машинах кількість кадрів за секунду може бути різною.

Кращий спосіб — задавати швидкість у пікселях за секунду і множити її на час одного кадру dt.

sf::Clock clock;

while (window.isOpen()) {
    float dt = clock.restart().asSeconds();

    position += velocity * dt;
}

Тоді рух буде більш однаковим на різних комп’ютерах.

9. Що ми будемо реалізовувати

  1. Кольорова кулька.
  2. Кулька рухається і відбивається від стін.
  3. Малюємо поле або стіни.
  4. Кулька змінює колір при відбиванні.
  5. Клавіатура змінює швидкість і напрям.
  6. Клік мишкою додає нову кульку.
  7. Кульки можуть зіштовхуватися.

10. Мінімальний результат заняття

Наприкінці уроку у вас має бути програма, у якій:

  • відкривається вікно;
  • є кулька;
  • кулька рухається;
  • кулька відбивається від меж поля;
  • кулька змінює колір при ударі;
  • швидкість або напрям змінюються з клавіатури.

11. Ідеї для самостійного розвитку

  • додати кілька кульок;
  • робити нову кульку кліком миші;
  • порахувати кількість відбивань;
  • зробити поле меншим за вікно;
  • додати текстовий лічильник;
  • зробити зіткнення кульок більш реалістичними.
Головне на цьому уроці — не кількість ефектів, а розуміння моделі: події, цикл, стан, оновлення, кадр.
