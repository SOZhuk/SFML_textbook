6. Навчальний проєкт: базова кулька

Починаємо з маленьких кроків. Після кожного кроку проєкт має компілюватися і запускатися. Якщо щось пішло не так, легше повернутися на один крок назад, ніж шукати помилку у великому файлі.

Працюємо в одному файлі src/main.cpp. Для першого графічного проєкту це нормально: головне зараз зрозуміти цикл програми, стан об'єкта і малювання кадру.

Далі проєкт буде рости поступово: від однієї кульки до поля, керування, кількох об'єктів і додаткових графічних ефектів. У кожному кроці спочатку є невелика мета, потім зміна в коді, проміжний результат і маленька перевірка. Не копіюйте великі готові приклади одразу: краще додавати одну зміну і одразу перевіряти результат.

Крок 0. Вікно і порожній цикл

Мета цього кроку - отримати стабільну основу програми. Почніть із мінімального коду: він тільки відкриває вікно і дозволяє його закрити.

#include <SFML/Graphics.hpp>
#include <optional>

int main() {
    sf::RenderWindow window(sf::VideoMode({800, 600}), "SFML lesson");

    while (window.isOpen()) {
        while (const std::optional event = window.pollEvent()) {
            if (event->is<sf::Event::Closed>()) {
                window.close();
            }
        }

        window.clear(sf::Color(30, 30, 30));
        window.display();
    }

    return 0;
}

Проміжний результат: відкривається темне вікно 800x600, кнопка закриття працює.

Спробуйте ще: змініть заголовок вікна або колір у window.clear(...).

Крок 1. Одна кулька

Мета цього кроку - намалювати перший об'єкт. Додайте кульку перед головним циклом:

sf::CircleShape ball(30.f);
ball.setPosition({100.f, 100.f});
ball.setFillColor(sf::Color::Green);

У частині малювання додайте:

window.draw(ball);

Повний порядок малювання має бути таким:

window.clear(sf::Color(30, 30, 30));
window.draw(ball);
window.display();

Проміжний результат: у вікні видно зелену кульку.

Спробуйте ще: змініть радіус, позицію або колір кульки.

Крок 2. Стан кульки

Мета цього кроку - підготувати дані, які будуть змінюватися під час руху. Поки що позиція і колір заховані всередині ball. Для руху зручніше явно зберігати стан у змінних.

Перед створенням кульки додайте:

float radius = 30.f;
sf::Vector2f position{100.f, 100.f};
sf::Vector2f velocity{0.8f, 0.6f};
sf::Color color = sf::Color::Green;

Створення кульки тепер можна записати так:

sf::CircleShape ball(radius);
ball.setPosition(position);
ball.setFillColor(color);

Проміжний результат поки не зміниться. Але код готовий до руху: у нас уже є позиція, швидкість, радіус і колір.

У цьому тимчасовому варіанті velocity означає "на скільки пікселів зсунути кульку за один кадр". Тому значення маленькі: якщо додати 160.f за кожен кадр, кулька майже одразу вилетить з вікна.

Спробуйте ще: змініть velocity і передбачте, у який бік мала б рухатися кулька.

Крок 3. Рух

Мета цього кроку - побачити найпростішу ідею руху: кожен кадр трохи змінювати позицію. У частині Оновлення стану додайте:

position += velocity;
ball.setPosition(position);

Тепер логіка кадру виглядає так:

// 1. Обробка подій.

// 2. Оновлення стану.
position += velocity;
ball.setPosition(position);

// 3. Малювання кадру.

Проміжний результат: кулька рухається, але з часом вилітає за межі вікна.

Важливе обмеження: такий рух залежить від кількості кадрів за секунду. Якщо комп'ютер малює більше кадрів, кулька робить більше зсувів за ту саму секунду і рухається швидше.

Спробуйте ще: зробіть рух повільнішим або швидшим, змінивши velocity.

Крок 4. Відбивання від меж вікна

Мета цього кроку - не дозволити кульці вилітати за межі вікна. Додайте сталі розміри вікна поруч зі змінними стану:

const float windowWidth = 800.f;
const float windowHeight = 600.f;

Після оновлення позиції додайте перевірку меж:

if (position.x <= 0.f) {
    position.x = 0.f;
    velocity.x = -velocity.x;
}

if (position.x + 2.f * radius >= windowWidth) {
    position.x = windowWidth - 2.f * radius;
    velocity.x = -velocity.x;
}

if (position.y <= 0.f) {
    position.y = 0.f;
    velocity.y = -velocity.y;
}

if (position.y + 2.f * radius >= windowHeight) {
    position.y = windowHeight - 2.f * radius;
    velocity.y = -velocity.y;
}

ball.setPosition(position);

У цьому варіанті position означає не центр кульки, а лівий верхній кут прямокутної області, у яку вписане коло. Тому правий край кульки має координату position.x + 2.f * radius, а нижній край - position.y + 2.f * radius.

SFML дозволяє змінити точку прив'язки об'єкта через setOrigin(...). Наприклад, якщо поставити origin у центр кола, то position можна буде читати як координату центра:

ball.setOrigin({radius, radius});

Тоді перевірки меж записувалися б інакше: лівий край був би position.x - radius, а правий - position.x + radius. У цьому проєкті залишаємо стандартну прив'язку до лівого верхнього кута, щоб усі наступні кроки використовували одну й ту саму модель координат. Якщо змінити origin зараз, доведеться змінювати і всі перевірки меж далі.

Проміжний результат: кулька рухається і відбивається від країв вікна.

Спробуйте ще: змініть радіус кульки і перевірте, чи вона все одно правильно відбивається.

Крок 5. Час кадру і обмеження FPS

Мета цього кроку - зробити рух незалежним від кількості кадрів за секунду. Для цього velocity буде означати не "пікселів за кадр", а "пікселів за секунду".

Спочатку обмежте кількість кадрів. Додайте після створення вікна:

window.setFramerateLimit(60);

Це просить SFML не малювати більше ніж приблизно 60 кадрів за секунду. Так програма не витрачає зайві ресурси, а рух стає легше спостерігати.

Тепер замініть тимчасову швидкість на швидкість у пікселях за секунду:

sf::Vector2f velocity{160.f, 120.f};

Додайте годинник перед головним циклом:

sf::Clock clock;

На початку кожного кадру, вже всередині while (window.isOpen()), отримайте час кадру:

float dt = clock.restart().asSeconds();

dt - це кількість секунд, яка минула від попереднього кадру. На 60 FPS це приблизно 0.016.

Після цього замініть простий рух:

position += velocity;

на рух з урахуванням часу:

position += velocity * dt;

Тепер velocity читається як "пікселів за секунду". Наприклад, 160.f по осі x означає приблизно 160 пікселів за одну секунду, навіть якщо кількість кадрів трохи змінюється.

Чому не просто sf::sleep(...)

Для навчального експерименту можна вручну зробити паузу в кінці кадру:

sf::sleep(sf::milliseconds(16));

Такий код приблизно чекає 16 мілісекунд, тобто близько одного кадру при 60 FPS. Але це не найкращий основний спосіб керувати рухом: реальний час кадру все одно може змінюватися, а ручна пауза не пояснює, скільки саме часу минуло.

У цьому проєкті краще поєднати два інструменти: window.setFramerateLimit(60) обмежує частоту кадрів, а dt робить рух незалежним від того, скільки часу реально минуло між кадрами.

Проміжний результат: кулька рухається і відбивається як раніше, але тепер швидкість описана в пікселях за секунду.

Спробуйте ще: тимчасово змініть window.setFramerateLimit(30) або window.setFramerateLimit(120) і перевірте, що швидкість руху лишається приблизно однаковою.

Контрольна точка

Після цієї сторінки у вас має бути програма, де:

  • відкривається вікно;
  • у вікні є кулька;
  • кулька рухається;
  • кулька відбивається від країв;
  • рух використовує dt і не прив'язаний напряму до кількості кадрів.

Це перший робочий результат. Далі будемо додавати колір, поле, керування і лічильники.

Повний код після кроку 5

Якщо ваша програма не компілюється або кулька поводиться не так, звірте свій src/main.cpp з цим варіантом.

#include <SFML/Graphics.hpp>
#include <optional>

int main() {
    sf::RenderWindow window(sf::VideoMode({800, 600}), "SFML lesson");
    window.setFramerateLimit(60);

    const float windowWidth = 800.f;
    const float windowHeight = 600.f;

    float radius = 30.f;
    sf::Vector2f position{100.f, 100.f};
    sf::Vector2f velocity{160.f, 120.f};
    sf::Color color = sf::Color::Green;

    sf::CircleShape ball(radius);
    ball.setPosition(position);
    ball.setFillColor(color);

    sf::Clock clock;

    while (window.isOpen()) {
        float dt = clock.restart().asSeconds();

        while (const std::optional event = window.pollEvent()) {
            if (event->is<sf::Event::Closed>()) {
                window.close();
            }
        }

        position += velocity * dt;

        if (position.x <= 0.f) {
            position.x = 0.f;
            velocity.x = -velocity.x;
        }

        if (position.x + 2.f * radius >= windowWidth) {
            position.x = windowWidth - 2.f * radius;
            velocity.x = -velocity.x;
        }

        if (position.y <= 0.f) {
            position.y = 0.f;
            velocity.y = -velocity.y;
        }

        if (position.y + 2.f * radius >= windowHeight) {
            position.y = windowHeight - 2.f * radius;
            velocity.y = -velocity.y;
        }

        ball.setPosition(position);

        window.clear(sf::Color(30, 30, 30));
        window.draw(ball);
        window.display();
    }

    return 0;
}
