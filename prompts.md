# Prefix Prompts

Use one of these before a vague request so the work stays safe and consistent.

## Content extraction

`Preserve existing HTML behavior, extract only the requested content, keep docs/ as output, and do not redesign the site.`

## Chapter split

`Split the chapter into source parts first, keep the published HTML stable, and update the build only after the source structure is clear.`

## Syntax highlighting

`Use the existing highlight.js assets, generate language-specific code blocks, and keep the legacy HTML book compatibility intact.`

## Minimal build

`Prefer the smallest static build path possible, avoid heavy frameworks, and keep all generated files compatible with GitHub Pages.`

## Backup safety

`Preserve a backup copy before rewriting any generated page, and document the change in meta/ after each iteration.`
