# 2. Створення та запуск першого проєкту SFML

Основний навчальний маршрут використовує репозиторій, підготовлений викладачем. У ньому вже є налаштування VS Code, тому не потрібно окремо налаштовувати CMake. Такий проєкт збирається динамічно, а вихідний файл програми займає значно менше місця.

Спочатку запустимо готовий проєкт, а в наступному розділі розберемо, що саме працює всередині.

## Основний маршрут: готовий проєкт

:::warning Увага
Перед клонуванням оберіть папку для проєкту зі шляхом без кириличних літер, наприклад `D:\Projects`. Якщо шлях містить кирилицю, інструменти збірки можуть не знайти файли або завершити компіляцію з помилкою.
:::

Клонуйте готовий  [репозиторій з шаблоном](https://github.com/SOZhuk/SFML_template):

```powershell
git clone https://github.com/SOZhuk/SFML_template.git
```

Якщо потрібна допомога з командами Git, скористайтеся [довідником](https://git-practicum-a22961.gitlab.io/git-practicum.html).

:::details Короткий довідник Git
Найчастіше в навчальних проєктах знадобляться такі команди.

```powershell
git config --global user.name "Ваше Ім'я"
git config --global user.email "you@example.com"
```

```powershell
git clone https://github.com/<користувач>/<репозиторій>.git
cd <репозиторій>
```

```powershell
git status
git add .
git commit -m "Короткий опис зміни"
git push
```

```powershell
git pull
git log --oneline
```

`git status` показує поточні зміни, `git add` готує їх до коміту, `git commit` зберігає зміни локально, а `git push` відправляє їх у віддалений репозиторій.
:::

Перейдіть у папку проєкту:

```powershell
cd SFML_template
```

Відкрийте її у VS Code:

```powershell
code .
```

Переконайтеся, що основний код доступний у `src/main.cpp`.
Після цього можна переходити до компіляції і запуску навчального проєкту. Вам достатньо натиснути F5.

Очікуваний результат: вікно SFML успішно відкривається, і ви готові переходити до технічних деталей роботи програми.

## Альтернативний маршрут: CMake

Для професійної розробки автори бібліотеки SFML рекомендують використовувати утиліту CMake і шаблонний репозиторій. Зауважте, що цей шлях потребує більше налаштувань.

### Встановлення CMake

CMake потрібен для того, щоб VS Code міг налаштувати та зібрати C++ проєкт. На Windows найзручніше встановити його через `winget`:

```powershell
winget install --id Kitware.CMake -e
```

Після встановлення закрийте і знову відкрийте термінал, а потім перевірте, що команда доступна:

```powershell
cmake --version
```

Якщо команда не знайдена, перезапустіть VS Code або комп'ютер, щоб оновився `PATH`.

### Розширення VS Code для CMake

У VS Code встановіть розширення **CMake Tools** від Microsoft. Його ідентифікатор у Marketplace:

```text
ms-vscode.cmake-tools
```

Через командний рядок його можна встановити так:

```powershell
code --install-extension ms-vscode.cmake-tools
```

Саме це розширення додає команди `CMake: Configure`, `CMake: Select a Kit`, `CMake: Build` і панель CMake у VS Code.

### Створення власного репозиторію

Відкрийте шаблонний репозиторій:

```text
https://github.com/SFML/cmake-sfml-project
```

На GitHub натисніть **Use this template** -> **Create a new repository**. Назвіть репозиторій, наприклад `sfml-first-project`, і створіть його у своєму GitHub акаунті.

Після створення скопіюйте адресу свого репозиторію. Вона матиме приблизно такий вигляд:

```text
https://github.com/<ваш-github-логін>/sfml-first-project.git
```

### Клонування проєкту

Перейдіть у директорію, де будуть зберігатися ваші навчальні проєкти. Наприклад:

```powershell
cd D:\Projects
```

Склонуйте свій репозиторій:

```powershell
git clone https://github.com/<ваш-github-логін>/sfml-first-project.git
```

Перейдіть у папку проєкту:

```powershell
cd sfml-first-project
```

Відкрийте її у VS Code:

```powershell
code .
```

### Налаштування CMake Tools

Коли VS Code відкриє проєкт, розширення CMake Tools може запропонувати обрати toolkit. У CMake Tools це називається **Kit**.

Якщо автоматичне вікно не з'явилося, відкрийте Command Palette:

```text
Ctrl+Shift+P
```

Запустіть команду:

```text
CMake: Select a Kit
```

Оберіть встановлений компілятор C++. Наприклад:

- `Visual Studio Build Tools` або `Visual Studio Community`, якщо працюєте з MSVC;
- `GCC` або `MinGW`, якщо використовуєте MinGW;
- `Clang`, якщо він встановлений окремо.

Після вибору Kit запустіть налаштування проєкту:

```text
CMake: Configure
```

### Перший build

Коли конфігурація завершиться без помилок, запустіть збірку:

```text
CMake: Build
```

Або скористайтесь кнопкою **Build** на CMake-панелі внизу VS Code.

Перший build може тривати довше, тому що шаблонний проєкт завантажує і збирає залежності. Якщо збірка завершилась успішно, можна запускати програму з VS Code.


## Підсумок

Після виконання одного з маршрутів проєкт має скомпілюватися, а під час запуску має відкритися SFML-вікно з кулькою: зеленою для CMake-маршруту, жовтою для власного репозиторію на основі підготовленого шаблону.
